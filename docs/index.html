Installation

It is recommended to use Composer to install Hyper.

composer create-project vulcanphp/hyper myapp

Then go to project root directory and run the development server using:

php -S localhost:8080 -t public

Then open: http://localhost:8080

Note: In a production server, make sure all HTTP requests are redirected to the public/ folder.

Application Configuration

Configure your application in the app/bootstrap.php file. Here is an example:

# app/bootstrap.php

use admin\admin;
use hyper\application;

return new application(
    path: __DIR__,
    routesPath: __DIR__ . '/web/routes.php',
    requirePath: [
        __DIR__ . '/web/providers.php',
        __DIR__ . '/web/middlewares.php',
    ],
    providers: ['setYourCustomProvider'],
    middlewares: ['csrfProtectionMiddleware'],
    env: [
        'lang' => 'en',
        'lang_dir' => __DIR__ . '/i18n',
        'tmp_dir' => __DIR__ . '/../public/tmp',
        'upload_dir' => __DIR__ . '/../public/uploads',
        'media_url' => '/uploads/',
        'asset_url' => '/resources/',
        'database' => [
            'driver' => 'sqlite',
            'file' => __DIR__ . '/../sqlite.db'
        ],
    ],
);

Manage Routes

Add routes to your application in the app/web/routes.php file. For example:

# app/web/routes.php

return [
    ['path' => '/', 'template' => 'welcome'],
    ['path' => '/about', 'callback' => fn() => 'About Us Page', 'name' => 'about'],
    ['path' => '/account', 'callback' => [account::class, 'index'], 'middleware' => ['checkLogin']],
];

Manage Views

Create view files in the app/views folder with the namespace views. For example:

# app/views/account.php

namespace views;

class account {

    public function index() {
        return template('account', ['name' => 'John Doe']);
    }

}

Manage Templates

Create templates in the app/templates folder. First, create a layout for all templates. For example:

# app/templates/layout.php

<html lang="en">
    <head>
        <title><?= $title ?></title>
    </head>
    <body>
        <?= $template->include("includes/header") ?>
        <?= $content ?>
    </body>
</html>

Create a header part in another subfolder called includes:

# app/templates/includes/header.php

<header>
    <h1>Welcome to Hyper PHP</h1>
</header>

Finally, create as many templates as required by your application. For example:

# app/templates/welcome.php

<?php
$template->layout("layout");
$template->set("title", "Welcome to Hyper");
?>

<h2>Hyper PHP is a Sweet and MVT Based PHP Tiny Web Framework</h2>

Manage Models

To create new models, go to the app/models folder and create a new model with the namespace models. For example: 

# app/models/student.php

use hyper\model;

class student extends model {
    protected string $table = 'students';

    public string $name;
    public int $age;
}

To perform database operation using model, here is a simple example: 

# app/views/home.php

use models\student;

// save a new student
$student = new student();
$student->name = 'Shahin Moyshan';
$student->age = 22;
$student->save();

// find a student model and update/remove it
$student = student::find(1);
var_dump($student);

// update student model
$student->age = 23;
$student->save();

// remove a student
$student->remove();

// list of all students
var_dump(student::get()->result());

// paginate with 20 students per page
$paginator = student::get()->paginate(20);
var_dump($paginator);

// get data from paginator
var_dump($paginator->getData());

// get links from paginator
var_dump($paginator->getLinks());

// filter students
var_dump(student::where('age >= 16')->result());

// usage of query builder from model
$query = student::query();

// insert records into `students` table using query builder
$query->insert(['name' => 'Shahin Moyshan', 'age' => 22]);

// delete records from `students` table using query builder
$query->delete(['id' => [1, 2, 3]]);

// update record into `students` table using query builder
$query->update(['age' => 23], ['id' => 1]);

// select records from `students` table using query builder
var_dump(
    $query->select('name, age')
        ->where('age >= 16')
        ->order('id DESC')
        ->limit(10, 20)
        ->result() // or ->paginate(20)
);

Manage Middlewares

To add a new middleware, open the app/web/middlewares.php file and create a function that will process HTTP requests. After creating the function, register it in the router middleware array or globally in the app/bootstrap.php file.

Example for the authMiddleware:

# app/web/middlewares.php

function authMiddleware(request $request) {
    if (!isset($request->user)) {
        return new response('Unauthorized', 401);
    }
}

Register the authMiddleware:

# app/web/routes.php

return [
    ['path' => '/account', 'middleware' => ['authMiddleware'], ...],
];

Or register globally:

# app/bootstrap.php

return new application(
    middlewares: ['authMiddleware'],
    ...
);

Service Providers

To add a new service provider, open the app/web/providers.php file and create a function that will manipulate the current application. Register the service provider in the app/bootstrap.php file.

Example for the csrfProtectionProvider:

# app/web/providers.php

function csrfProtectionProvider(application $app) {
    if (!$app->session->has('_token')) {
        $app->session->set('_token', bin2hex(random_bytes(32)));
    }
}

Register in the bootstrap file:

# app/bootstrap.php

return new application(
    providers: ['csrfProtectionProvider'],
    ...
);

Manage Admin

You can manage your website data under the models in the app/models folder. Register the models in the admin panel to easily process CRUD operations. You can also create custom menus with dynamic output.

In the bootstrap.php file, the admin provider and customization are added like this:

# app/bootstrap.php

use admin\admin;
return new application(
    providers: [[new admin(), 'setup']],
    env: [
        'admin' => __DIR__ . '/web/admin.php',
        'admin_prefix' => 'admin'
    ],
    ...
);

Open the app/web/admin.php file to manage the admin panel. Example:

# app/web/admin.php

use models\department;
use models\student;
use admin\core\modelView;
use views\backup;

return [
    // Set up your admin panel login user
    'user' => [
        'name' => 'admin', // Whatever name you want
        'password' => 'admin' // Replace with a strong password in production
    ],
    'models' => [
        department::class, // Add a basic model
        new modelView(
            model: student::class,
            name: 'student',
            name_plural: 'students',
            fields: ['id', 'name', 'age', 'department'],
            search: ['name'],
            filter: ['gender' => ['M' => 'Male', 'F' => 'Female']],
            with: ['department'],
            where: ['deleted' => false],
            order: 'id DESC',
            actions: ['Export Selected Students Result Sheet' => url('admin/student/export')]
        ), // Add a customized model
    ],
    'menus' => [
        // Add a menu with views which will return a string
        'backup' => [backup::class, 'index'],
        // Add another menu with a file to get a string output
        'cms' => __DIR__ . '/../cms/index.php',
        // Add a simple menu with a hardcoded string
        'api' => 'sakjadasdas5d1as54d5asd2asd1ds88sdf'
    ],
    'settings' => [
        // Register all grouped settings
        'general' => [
            // Register inputs for this group.
            ['type' => 'text', 'name' => 'title']
            ['type' => 'file', 'name' => 'logo']
        ],
    ],
];

Site Settings

You can get setting value that registered into app/bootstrap.php file under settings array. 

// get a setting with default value
var_dump(setting('general', 'title', 'My Default Title'));

// get settings instance
var_dump(settings());

// check if setting is defined or not
if(settings()->has('general', 'title')) {

}

// change/update setting value
settings()->set('general', 'title', 'My New Title');

// remove a setting
settings()->remove('general', 'title');

// bulk update multiple settings
settings()->setup('general', [
    'title' => 'My New Title',
    'logo' => './new/path/logo.png'
]);

Vite Usage

Hyper comes with Vite and Tailwind CSS built-in. Run the following commands to use Vite with Hyper.

Install npm and run Vite development:

npm install && npm run dev

Include the Vite script in your template. For example:

# app/templates/layout.php

<html lang="en">
    <head>
        <?= vite(["running" => true]) ?>
    </head>
    <body>
        ...
    </body>
</html>

To run production for Vite:

npm run build

Change the Vite running status in the PHP template to false:

# app/templates/layout.php

<html lang="en">
    <head>
        <?= vite(["running" => false]) ?>
    </head>
    <body>
        ...
    </body>
</html>

**Hyper Core

HyperCore is the backbone of the Hyper MVT Framework, providing essential core classes, utility functions, and helper methods to streamline web development. This document details all available classes and functions within the HyperCore.

Core Classes

Application

  • Class: application
  • Description: Application container that manages the overall lifecycle of your application.

Database

  • Class: database
  • Description: PDO database container for managing database connections and operations.
  • Example: 
    use hyper\database;
$database = new database([
    // sqlite, mysql, pgsql, cubrid, dblib, firebird, ibm, informix, sqlsrv, oci, odbc
    'driver' => 'sqlite',

    // required when driver is sqlite
    'file' => __DIR__ . '/../sqlite.db',

    // pre-define a custom dsn for pdo else, it will auto genere a dsn from this config array.
    'dsn' => null,

    // define your custom pdo options. Note: PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION added as default.
    'options' => [],

    // when driver is serverside
    'host' => 'localhost',
    'user' => '{user}',
    'password' => '{password}',
    'port' => 3306,
    'name' => 'dbname',
    'charset' => 'utf8mb4'

    // Learn more: https://www.php.net/manual/en/book.pdo.php
]);
dump($database->prepare('SELECT * FROM students'));

Request

  • Class: request
  • Description: HTTP request class for handling and processing incoming requests.

Response

  • Class: response
  • Description: HTTP response class for managing outgoing responses.

Middleware

  • Class: middleware
  • Description: Middleware class for handling HTTP request routing.
  • Example: 
    use hyper\middleware;

$middleware = new middleware();
$middleware->add(callback);
$middleware->handle(request: $request);

Router

  • Class: router
  • Description: Router class for defining and handling application routes.
  • Example: 
    use hyper\router;

$router = new router(middleware: $middleware);
$router->add('/', fn() => 'Hello World');
$response = $router->dispatch();
$response->send();

Model

  • Class: model
  • Description: Model class for handling database interactions with ORM and form handling.
  • Example: 
    use hyper\model;

class student extends model {
    protected string $table = 'students';

    public string $name;
    public int $age;
    public string $department;
}

dump(student::get()->result());

Query

  • Class: query
  • Description: PHP PDO query builder for constructing and executing database queries.
  • Example: 
    use hyper\query;
$query = new query(database: $database, table: 'students');
dump($query->result());

Session

  • Class: session
  • Description: Session class for managing user sessions.

Template

  • Class: template
  • Description: Template engine class for rendering views.
  • Example: 
    use hyper\template;
$template = new template(__DIR__);
dump($template->render('welcome', ['message' => 'Welcome to App']));

Translator

  • Class: translator
  • Description: Translator helper class to retrieve translated text.

Utility Classes

Cache

  • Class: cache
  • Description: Cache class for caching data.
  • Example: 
    use hyper\utils\cache;
$cache = new cache('home');
dump($cache->load('time', fn() => 'Now :' . time(), '5 minutes'));

Collect

  • Class: collect
  • Description: Collection class for handling arrays.

Form

  • Class: form
  • Description: Form builder class for constructing and managing forms.
  • Example: 
    use hyper\utils\form;

// independent usage
$form = new form(request: $request, fields: [['type' => 'text', 'name' => 'name']]);
// usage from model
$form = new form(request: $request, model: $student);

// add new form field
$form->add(['type' => 'email', 'name' => 'email', 'required' => true]);

// validate form
if($form->validate()){
    dump($form->getData());
}else{
    echo $form->render();
}

Hash

  • Class: hash
  • Description: Class for hashing and encryption.

Image

  • Class: image
  • Description: Image helper class for managing image operations.
  • Example:

$image = new image(__DIR__ . '/image.png');
$image->compress(50);

Paginator

$paginator = new paginator(total: 500, limit: 20);

Ping

  • Example:

Uploader

  • Class: uploader

  • Example:

Validator

  • Class: validator
  • Description: HTTP input validator class.
  • Example: 
    use hyper\utils\validator;
$validator = new validator();
dump($validator->validate([
    'name' => ['required', 'min:10', 'max:60'],
    'email' => ['required', 'email'],
], $request->all()));

Helper Classes

Form

  • Class: form
  • Description: Trait for model to extract form fields from model object properties.
  • Example: 
    use hyper\model;
use hyper\helpers\form;

class student extends model {
    use form;

    protected function form(): array {
        return [
            'name' => ['type' => 'text', 'required' => true],
            'email' => ['type' => 'email', 'required' => true],
            'gender' => ['type' => 'radio', 'options' => ['M' => 'Male', 'F' => 'Female']],
        ];
    }
}

$form = new form(request: $request, model: new student());
dump($form->render());

Mail

  • Class: mail
  • Description: PHP built-in mail class.
  • Example: 
    use hyper\helpers\mail;
$mail = new mail();
$mail->address('john.doe@main.com', 'John Doe');
$mail->replyTo('shahin.moyshan2@gmail.com');
$mail->subject('Test Mail');
$mail->body('Hello World, This is Test Mail From Shahin Moyshan');
$mail->send();

ORM

  • Class: orm
  • Description: Object-Relational Mapper for database interactions.
  • Example: 
    use hyper\model;
use hyper\helpers\orm;

class student extends model {
    use orm;

    protected function orm(): array {
        return [
            'department' => ['has' => 'one', 'model' => department::class, 'lazy' => false],
            'subjects' => ['has' => 'many-x', 'model' => subject::class, 'table' => 'students_subjects'],
            'results' => ['has' => 'many', 'model' => result::class, 'formIgnore' => true],
        ];
    }
}

dump(student::with(['subjects', 'results', 'department'])->paginate(20));

Uploader

  • Class: uploader
  • Description: Uploader helper for managing model uploads.
  • Example: 
    use hyper\model;
use hyper\helpers\uploader;

class student extends model {
    use uploader;

                'resize' => [540 => 540],
                'resizes' => [140 => 140, 60 => 60]

}
